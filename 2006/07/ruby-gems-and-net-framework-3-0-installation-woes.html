    Regarding ruby gems, if you were getting this error:

    ERROR: While executing gem … (Gem::GemNotFoundException)

    Jim Weirich posted this solution on ruby-talk earlier this week:

    There was a gem on rubyforge that somehow corrupted the gem index on that site. A lot of people seemed to have downloaded the corrupted index and it has been causing problems. The easiest way to deal with it is to just delete your gem cache file.

    Use the gem env command to find the default gem repository directory and remove the source_cache file there.

    $ gem env gemdir
PATH_TO_DEFAULT_GEM_REPOSITORY
$ rm PATH_TO_DEFAULT_GEM_REPOSITORY/souce_cache
$ rm $HOME/.gem/source_cache

    The .gem cache will only exist if the default repository is not writable from your user account. Delete both if they exist.

    Regarding .NET Framework 3.0, I was getting a really bizarre exception about mismatched library versions when trying to display images using WPF. It turns out the culprit is the Windows Imaging Components. You’ll need to update via this link.
