Xen to be the new kernel of OS X?

Posted by John Lam Sat, 04 Feb 2006 15:52:00 GMT

I saw this post from Colin, the guy who runs the winxponmac.com contest site. Apparently, he talked to one of the guys from XenSource, and this was the most interesting parts of the conversation:

Currently, Apple uses the Mach Kernel to interface with the hardware. However, “Mach” is essentially outdated and causes a lot of the bottlenecks with programs having to create a lot more processes than necessary to get the job done. The guy from Xen told me they have been talking a great deal with the Apple guys about replacing Mach with Xen. This is a really big deal! This would be consistent with reports that Apple has filed patents related to Virtualization.

Read the rest of the post for more details (and links). Xen is an open-source virtualization monitor a’la VMWare and VPC. It apparently is much more efficient than either VMWare or VPC when the guest OS has been modified to run on Xen. However, with the advent of the new Intel Vanderpool virtualization technology, the guest OS can run unmodified at Ring 1 privilege. Now, it’s likely that both VMWare and VPC will exploit this new hardware feature, but it would rock if we could have built-in virtualization in the host OS.
