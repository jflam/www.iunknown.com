Hardware changes

Posted by John Lam Tue, 07 Feb 2006 17:09:00 GMT

I’ve had it with HDD failures on workstations. I didn’t suffer much in the way of data loss, it’s just the freaking inconvenience of it that drives me crazy.

I just finished building a new workstation that runs on an Opteron 175 processor. I thought I’d collect my observations on this process here for folks who may want to build something on this platform in the future.

First off, I didn’t want SLI, so I didn’t want to pay for it. I elected to purchase an nForce4 chipset ASUS A8N E motherboard instead of one of the nForce4 SLI chipset motherboards. That was a minor mistake. This motherboard cannot boot from an external USB drive (which is a pain), nor does it have any Firewire ports (long story short, my 3G iPod doesn’t sync to USB 2 – only Firewire for some bizarre reason). I didn’t figure this out of course until I had the entire computer assembled, and by that time I’d be dammed if I was going to rip it apart to replace the motherboard! Bottom line: don’t skimp on the mobo (or at least read and memorize the specs before you buy!)

I’m now running RAID 0+1 across 4×250GB 7200rpm disks. I hope that the chances of HDD failure causing downtime on that system are a thing of the past. My primary server runs a RAID5 array with 4×160GB disks, so I’ve always had redundancy there (but I haven’t suffered a failure on those drives yet either). But I’ll now have performance and redundancy with RAID 0+1 on the workstation.

A warning for folks who are running nForce RAID and want to install Vista – it won’t work. Apparently the February CTP will fix this issue. I might drop Vista onto a vmware image just to see how it works – it’s been a while since I’ve looked at that OS. I’d love to see it run native though :)

I’m amazed by how responsive this system is – it feels more responsive than the Pentium D820 that it replaced. I was doing a lot of multitasking when I was rebuilding my image, and it just felt like there was always more cycles available.

The Opteron 175 ran really cool during the things I was doing (SpamBayes training, installing VS 2005, lots of downloading simultaneously). The CPU utilization was consistently > 80%, yet the CPU temperature monitor never went above 42C the entire time.

It’s quiet too: I’m housing everything in a Sonata II case, and even 4×7200 rpm disks under my feet don’t seem to make much of a racket at all – it’s only a bit louder than the Shuttle SB31P that it’s replacing.

Bottom line: I’m happy with how quickly things came together from my favorite hardware suppliers: ASUS + Antec.

Xen to be the new kernel of OS X?

Posted by John Lam Sat, 04 Feb 2006 18:52:00 GMT

I saw this post from Colin, the guy who runs the winxponmac.com contest site. Apparently, he talked to one of the guys from XenSource, and this was the most interesting parts of the conversation:

Currently, Apple uses the Mach Kernel to interface with the hardware. However, “Mach” is essentially outdated and causes a lot of the bottlenecks with programs having to create a lot more processes than necessary to get the job done. The guy from Xen told me they have been talking a great deal with the Apple guys about replacing Mach with Xen. This is a really big deal! This would be consistent with reports that Apple has filed patents related to Virtualization.

Read the rest of the post for more details (and links). Xen is an open-source virtualization monitor a’la VMWare and VPC. It apparently is much more efficient than either VMWare or VPC when the guest OS has been modified to run on Xen. However, with the advent of the new Intel Vanderpool virtualization technology, the guest OS can run unmodified at Ring 1 privilege. Now, it’s likely that both VMWare and VPC will exploit this new hardware feature, but it would rock if we could have built-in virtualization in the host OS.
