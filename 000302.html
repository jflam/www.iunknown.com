What does "Enterprise" mean?

The term "Enterprise" seems to have many meanings. Here's my stab:

  • Enterprise applications are much more expensive
  • Enterprise applications are used by more than one person concurrently
  • Enterprise applications are typically browser-hosted today
  • Enterprise applications solve business problems1
  • Enterprise developers need to use more expensive tools

But I don't think that Enterprise applications are necessarily any more complex than non-Enterprise applications. I don't think that anyone would call a C++ compiler an Enterprise application, yet I would argue that it is far more complex than most Enterprise applications today.

From a developer standpoint, do you really need to use Enterprise tools (read: more expensive) to build Enterprise applications? I use VS.NET Enterprise Architect because it's the most expensive version of VS.NET that comes "for free" with my MSDN Universal subscription. I know lots of folks who install Windows 2003 Enterprise Server because itâ€™s the most expensive version of Windows (I know I did). I know it makes us feel better to use the more expensive tools, but do we really need to use any of the more expensive features?

The reason why I'm writing this is due to my frustration with tool vendors that only seek to sell expensive bundles of software to "Enterprise" developers. As an example, I can't buy a copy of NuMega's SoftICE debugger without buying their SoftICE Driver Suite product which costs $999!. Now, I don't write device drivers, but I could use a proven kernel-mode debugger every now and then. NuMega has lost me as a potential customer, and I'm thankful that MS gives away WinDbg for free for the odd time that I need to use a kernel-mode debugger.

[1] As opposed to technology problems, although you can argue that some business problems ARE technology problems.
