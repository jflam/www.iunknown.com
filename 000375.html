C++ programmer ethos

Why is it that whenever I start writing [Managed]C++ code that I:

  1. Feel the urge to write wrapper classes
  2. Worry much more about failure conditions / scenarios
  3. Think obsessively about const-correctness

Well, writing wrapper classes is pretty much all that I use MC++ for these days. There's a bunch of unmanaged Win32 API's that I need to call from managed code and MC++ is the perfect tool for the job (only a masochist would try and manipulate SIDs from C#/VB.NET).

Yes, I know that Whidbey provides wrappers for SIDs and their friends in System.Security.AccessControl, but I need to deploy systems today on V1.1.
