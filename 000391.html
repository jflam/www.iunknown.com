Ant pushback

I recently came across two rants about Ant. These rants were written by folks that I respect, and I'm quite disappointed that they didn't post more reasoned/detailed arguments.

The first, written by Bruce Eckel a confusing rant at best. I almost couldn't find his references to Ant amidst some reminiscing about C++ and the origins of Java. Bruce doesn't like Ant because: the file format is XML, and developers shouldn't be forced to have to write verbose build.xml files by hand. To me, this is largely an IDE problem as opposed to a problem with Ant as a build tool in and of itself. It isn't clear from looking at his rant what scenarios he had in mind when he wrote that "Ant sucks on a number of levels". It would be a much more informed discussion if he could point out specific deficiencies in Ant that caused him problems. I'm sure that the community would be more than happy to help him succeed with Ant.

The second, written by Martin Fowler is not particularly dismissive of Ant. Instead, he focuses on how conditionals and declarative languages don't mix. He proposes using an extension to a full-blown programming language, Ruby, as a replacement for Ant. I'd be interested in seeing some specific scenarios where he requires the power of a full-blown programming language to implement a build script.
