Serialization

There's some interesting XML serialization functionality built into Cocoa. You can write some fairly simple code such as:


NSMutableArray *array = [[NSMutableArray alloc] init];
[array addObject:@"John"];
[array addObject:@"Carolyn"];
if( [array writeToFile:@"people.plist" atomically:YES] )
  NSLog( "succeeded" );


which results in the following XML being produced:


...
<array>
  <string>John</string>
  <string>Carolyn</string>
</array>


Apparently Mac OS X uses XML configuration files known as plist files to configure functionality. There is a simple tool called the Property List Editor for editing these files.


I don't yet know how to serialize arbitrary objects (I tried serializing an NSArray of ABPerson objects, and that didn't work). Unfortunately, writeToFile failed silently without giving me a clue about what was causing it to fail.


The one thing that I am missing in Cocoa so far is a stream-based I/O model. It seems a bit crufty to have to send messages like writeToFile instead of writeToStream.
