Who's the Jeffrey Richter of the Mac OS X world?

As my anticipation is building for the newest member of my (computer) family, I have a bunch of questions that I would love to have answered by an "Advanced Windows" style of book, but for Mac OS X. As a long-time Windows dev, I have a lot of questions to ask about OS X:



  • How is code packaged for execution and sharing (i.e. what are the equivalents of EXE's and DLL's in OS X)

  • Is OS X a shell on top of Free BSD? In other words, I'm assuming that it is the Unix definitions of processes and threads that I should be reading about?

  • While looking at some Emacs distributions for OS X, I noticed that there isn't binary compatibility between OS 10.1 and OS 10.2. Why did the executable file format change? Is there an equivalent of Portable Executable files in that world?

  • Should I learn Cocoa? Or are there other ways of programming the Aqua UI?

  • How do I interoperate between code written using Objective-C and classic C/C++?

So many questions and so few books. My local bookstore has very few books on Mac OS X programming. I'm hoping that the resources on Apple's Developer Connection web site will help jump start my experience.

Posted by: Dave Lee at December 18, 2002 02:12 PM

1) How is code packaged for execution and sharing (i.e. what are the equivalents of EXE's and DLL's in OS X)
Mac OS X applications are actually a directory with a .app extension that the Finder and UI interpret as an application. Sharing code, ala DLLs, can be accomplished in a number of ways: one is to use the CFBundle framework to package the code as a bundle and load and call it dynamically. The other is to use the CFPlugIn APIs which are somewhat COM-compatible. You can also create low-level dynamic librarys, dylibs.
2) Is OS X a shell on top of Free BSD? In other words, I'm assuming that it is the Unix definitions of processes and threads that I should be reading about?
You should start by reading up on the Mach kernel which will give you an idea of processes and how threads work. There are a few different threading models available on Mac OS X (Carbon cooperative threads, pthreads, etc.).
3) While looking at some Emacs distributions for OS X, I noticed that there isn't binary compatibility between OS 10.1 and OS 10.2. Why did the executable file format change? Is there an equivalent of Portable Executable files in that world?
Not true...you can compile an app on Mac OS X 10.1 and it will run just fine on Mac OS X 10.2. Apple upgraded the gcc for 10.2 which doesn't produce backwards compatible ABI layout for 10.1.
4) Should I learn Cocoa? Or are there other ways of programming the Aqua UI?
Learn Cocoa. Learn Objective-C. You'll love it. I waited too long before diving into Objective-C cuz I was a fraidy cat and the square brackets looked intimidating. The programming is great and Cocoa frameworks are to die for. That said, you can create UIs in Interface Builder and code in Carbon or Cocoa-Java. Or, you can go 100% pure Java and use AWT and Swing.
5) How do I interoperate between code written using Objective-C and classic C/C++
Objective-C is C so you can mix the two to your heart's content -- I do it all the time. I call Carbon functions from Cocoa and you can even use Objective-C++ and get wicked cool integration between your C++ objects and Objective-C objects. Until you really start using the Mac OS X development environments, and figure out all of the options, its coolness is only something you read about!
