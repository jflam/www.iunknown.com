First Look at Venus

I spent some time this morning taking my first look at Venus (ASP.NET 2.0). The first thing that I wanted to pull apart was the new "Code Separation" compilation model that replaces "Code Behind" in ASP.NET 1.0/1.1.

Observation 1: Development uses the "Cassini" managed web server to host ASP.NET, not IIS. This is great for a number of reasons. Perhaps most important, I can now do ASP.NET development under a restricted account without having to jump through various hoops.

Observation 2: The code that ASP.NET generates is no longer in the usual place. Instead, you can find it under Local Settings\Temp\Temporary ASP.NET Files\... A big thanks goes out to Mark Russinovich's most excellent Filemon tool.

Observation 3: ASP.NET now uses the partial classes feature of C# for all user-defined code. When you write event handlers, the only code that you see is the code that handles the event. None of that nasty region-hidden code ever appears.

Observation 4: Intellisense works right away in the partial class file. This is really cool. Consider the following code frag:


using System;

namespace ASP
{
  public partial class Default_aspx
  {
    void Button1_Click(object sender, System.EventArgs e)
    {
      Label1.Text = "Clicked, me!";
    }
  }
}

When I type Label1 I see the Intellisense right away for the Label1 control. This requires a lot of background compilation goodness. Kudos to the IDE team for making this such a painless process.

Observation 5: Intellisense works in the .aspx file. Now, most of the time I don't see a good reason to put code in the .aspx file, but in those cases where you need to do this (customization at a customer site comes to mind) it sure helps having Intellisense in your <script runat="server"> blocks.

More observations later, but so far I'm really liking what I see.
