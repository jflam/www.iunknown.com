Interface Builder Rocks!

Interface Builder is an awesome forms designer package. I really like the way it understands the Aqua Human Interface Guidelines. For example, if you drag a UI element like a text field close to the edge of the form, IB will draw a dotted blue line that tells you where the boundary lies. This makes it really hard to create form-based applications that don't look "Mac like". Another cool feature happens when you select a UI element, hold the option button down, and roll your mouse pointer over another UI element. IB will tell you how far away that other element is on the screen. This makes it really easy to control how far apart items are spaced, as well as doing other operations like centering an item relative to another element.


I know I really need to get some screenshots to show this stuff in action. The problem is getting the hang of the workflow of doing a screen capture on my TiBook and then getting it over to my Windows box so that I can blog it (I'm still using CityDesk to do all of my blogging). What I really need to be able to do is blog from my TiBook ... but I'm not ready to go through the gyrations involved in switching my blogging software right now. There are just too many other cool things to do ... :)

When you do move your blog software over, I definitely recommend MovableType. I've been running on my TiBook with MySQL and it's been just great.
If you want to have some more fun with the Mac OS X developer tools, check out the WSMakeStubs tool in /Developer/Tools. It will read in a WSDL file from a local reference or a URL and generate SOAP proxy stubs in C/C++, Objective-C or AppleScript. Way, way, way cool. Hopefully it'll be rolled into Project Builder at some point with a nice UI and other options. But for now, it's pretty sweet.
