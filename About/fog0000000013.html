My publications
2001-12-02

I have co-authored two very different books, approximately 4 years apart.

The first book that I co-authored was the Delphi Developer's Handbook. I wrote it with Marco Cantu and Tim Gooch:


 
Amazingly enough, Amazon is still selling used copies of this book. It was written in 1997, and mostly refers to Delphi 3.

The second book that I co-authored was Essential XML. I wrote it with Don Box and Aaron Skonnard:


This was back in the day when I was very into XML. Now, I look at it as just another (useful) technology in my toolbox. There are more interesting problems to solve than the interop problem, although I'm sure that some others would disagree with me ;)

This is my attempt at generating a comprehensive list of all of my on-line publications.

  1. Microsoft IIS, COM+ and ASP. May 22, 2001. This was the one and only review that I have ever written in a print publication. It was fun doing it once, but I don't think I'll ever do it again ...
  2. Secure your Web Applications. PC Magazine, May 8, 2001. I wrote this piece for PC Magazine to discuss four different attack scenarios for your web site: a) Can I see what your visitors are seeing? b) Can I impersonate your visitors? c) Can I impersonate you? and d) Can I get your computers to do my bidding?
  3. One Session Per User. PC Magazine, May 8, 2001. I wrote this piece to discuss a specific session management problem that is common to web applications: how do I detect whether the same user is logged in multiple times to my web application?
  4. Auto-refreshing. PC Magazine, April 3, 2001. I wrote this piece to explain to developers how to get their browser to automatically refresh content.
  5. XSL Transformations. MSDN Magazine, August 2000. This is an excerpt from Essential XML where we discuss some possible applications for XSLT in transforming incompatible XML schemas. Since this time, I've revised my position on XSLT and XML documents: XSLT is truly only useful for transforming XML documents into HTML. This will be the subject of a future rant on this site.
  6. Architecting your Web Applications, Microsoft Internet Developer Magazine, September 1999. This article presents a somewhat radical view of building web applications: create them as a single page that is downloaded to the browser, and use it as the session state manager. Since this time, I've also revised my position on this technique: it's too difficult to implement for most developers. Also, recent technology developments such as ASP.NET make it far easier to pass session state as part of a page's call context.

There's quite a few more articles to add to the above list ... but Google can probably find them faster for you in any event :)
