AspectJ makes me want to write Java code again

Gregor Kiczales is a master at teaching. He gave the final keynote at Middleware 2004, and oh what a keynote it was. Anyone who teaches for a living should study how Gregor delivers a lecture. Idea convergence

I really love it when I can see existing ideas in a new light. Several ideas converged at Middleware 2004. Most of these ideas had come across my radar recently as a result of trying to expand my horizons. In no particular order, here's a few cool things that happened:

I met Steve Vinoski when we played the role of "industry" in a panel titled Is Middleware Dead?. Steve's an old-school CORBA guy recast as a web services guy. Later on, we discovered that we were on the same languages trajectory over the past few months. Both Steve and I had largely derided dynamically typed languages before falling in love with them in the past few months. There was lots of "dude! that's exactly the way I feel!" comments as we recounted our personal experiences.

During Werner Vogel's keynote, I ran across another concept that had particularly resonated with me just a week ago. Sean McGrath had talked about massive parallelization of XML transformations. The core idea was that you should emphasize ease of construction / maintenance over per-node performance if you could perform transformations in parallel. This idea was one of the concepts that Werner discussed in the Amazon architecture. Amazon doesn't particularly care how quickly a process runs per-node. It only matters whether they can run it in parallel. And since Moore's Law is rapidly driving the cost of computing to zero, it makes sense to leverage compute cycles over human cycles which don't benefit from Moore's Law scaling.

A few weeks ago, I attended the Python user group meeting here in Toronto. There, I learned about a company called Givex, and how they used Postgres database replication to add inexpensive fault-tolerence to their e-commerce application. At the Middleware conference, I met Bettina Kemme when she appeared on the opposite side of the panel that Steve and I were on. Bettina's research group is working on an implementation of Postgres replication in the database engine itself. I had previously never thought about using replication because of the lack of hard consistency guarantees. However, I'm now realizing that there are some very interesting projects going on in both academia and industry in this area, and that there's lots of opportunities to learn new things and grow.

That was easily one of the best $700 that I've spent in my career. Thanks to everyone who made it such a memorable event!

Middleware 2004 and Amazon.com

Werner Vogels gave an excellent keynote to open Middleware 2004. There was a lot of good keynote stuff in there- a great blend of motivation, information and insight. Werner recently accepted the position as Director of Systems Research at Amazon. Werner’s also looking for a few good people as well to work at Amazon.

He outlined the vision of Amazon.com, and what an ambitious vision it is. Amazon.com is implemented as a set of autonomous services. You know the sales rank for a book that takes up one tiny part of an Amazon page? That’s implemented as a separate cluster within an Amazon data center! Amazon.com has created a platform for experimenting with (and making money from) services. The Amazon platform is used by other companies already, including Nordstrom and Target department stores.

Amazon has problems moving to massive scale that other companies just don’t encounter. For example, some of Amazon’s data centers are only half-full. You know why? It’s because they simply cannot get any more POWER to those data centers. As a result, they are forced to move data between data centers which adds to complexity (and cost) of the system.

Update (to reflect the fact that eBay does this stuff and not Amazon): He also mentioned some interesting problems that eBay is having. They’re also pushing databases beyond what they are capable of. eBay does not maintain integrity constraints in the database – they’re maintained at the application layer. They don’t maintain indexes in the databases; instead lookups are done in Berkeley DB indices since lookup speeds are an order of magnitude faster there than on a relational database. Effectively, eBay is using their expensive database system as a transactional file system!

Fun with CodeDom

I decided to write a simplified version of Jeff Keys' most excellent Snippet Compiler to facilitate some experiments that I've been working on today. I absolutely love how easy it is to compile in-memory assemblies using System.CodeDom and friends:


private CompilerResults Compile()
{
  CodeDomProvider provider = new CSharpCodeProvider();
  ICodeCompiler compiler = provider.CreateCompiler();
  CompilerParameters parameters = new CompilerParameters();
  parameters.WarningLevel = 4;
  parameters.GenerateInMemory = true;
  parameters.GenerateExecutable = false;
  parameters.ReferencedAssemblies.Add("System.Web.Services.dll");
  parameters.ReferencedAssemblies.Add("System.Xml.dll");
  parameters.ReferencedAssemblies.Add("System.dll");
  return compiler.CompileAssemblyFromSource(parameters, textBoxSource.Text);
}

Richard Gabriel on Art in Programming

I spent some time reading some of Richard Gabriel's writings this evening. Much like Paul Graham's writing's that draw parallels between painting and programming, Richard Gabriel's writings draw parallels between poetry and programming.

One idea that caught my attention was the role that ambiguity plays in programming. He highlights the tension between being the desire to be well-defined and the desire for expressiveness. He uses polymorphism in object-oriented programming languages as an example:

However, polymorphism in object-oriented languages—especially those with extensive dynamic method-dispatch support—is an example of the use of (seemingly) ill-defined expressions for great expressive power. One could argue that a method-call is really quite well defined because, after all, only one class or object is chosen to respond, but the programmer in writing the method call cannot tell which piece of code that will be, so that it is sensible to think of the programmer as using ambiguity at code-writing time to gain clarity at code-execution time. In this sense, the artistic notion of ambiguity has at least inspired a breakthrough in programming language design.

He mentions aspect-oriented programming in passing as well. I wonder if my (current) bias against AOP is the result of my preference for well-defined behavior over expressiveness. I wonder if my current infatuation with Python might lead me down some new roads of thought as well ...

Power Surges and WinXP SP2

I'm not sure whether this is a good thing or not, but SP2 adds power surge detection on USB ports to the mix. My external USB hard drive is a Fujitsu 5400rpm drive that is rated at 0.55mA. Apparently the power limit on the Intel USB hub on my Inspiron 8600 is 0.5mA. So, after a few minutes of operation, it detects a surge on the port and shuts down the device.

If you use an external HDD to run Virtual PC instances, this can um ... cause undesirable effects :) So what you need to do is disable USB error notifications via Device Manager.

As I said before, I'm not sure whether this is a good thing or not; I'm hoping that the 0.05mA difference in power rating between my drive and the USB hub limit doesn't do anything bad. This is another reason why you should not put 7200 rpm drives into external USB 2.0 enclosures. The Hitachi drives that I looked at all drew > 1A of current, which is probably asking for trouble :)

Update: To properly workaround this problem, I bought a powered external USB 2.0 hub to provide power for my external HDD.

Lightweight Code Generation and Python

Recently on the IronPython mailing list, there has been discussion about the new Lightweight Code Generation (LCG) feature in V2.0 of the CLR. This feature is implemented by System.Reflection.Emit.DynamicMethod.

Joel Pobar wrote an entry some time ago (March 21, 2004) that describes how to write "Hello, World" using DynamicMethod. I recall reading this entry and saying to myself: "so what?". At the time, I didn't care much about dynamically typed languages that support eval-style programming. So I missed the significance of one bullet point:

Generated methods are able to be reclaimed.

Whoa. This changes everything for dynamically typed languages such as Python on the CLR. It is highly unlikely that future versions of the CLR will support unloading of assemblies from an existing AppDomain. This means that long-running programs that generate code via eval will eventually run out of memory since none of the generated code can be reclaimed by the GC. With LCG, however, dynamically generated methods can be reclaimed by the GC which is a big win for dynamic typing on the CLR.

Jim Hugunin the creator of IronPython and a recent hire at Microsoft wrote the following in a recent post:

LCG is the right way to fix this problem in the near term. Once again, I'll send you off to one of my co-worker's blogs for a concrete example: http://blogs.msdn.com/joelpob/archive/2004/03/31/105282.aspx

It should be a trivial matter to support this form of generation as an option in IronPython that is used only on systems that support DynamicMethod. This would mean that 1.1 systems would have an interactive interpreter with a memory leak, but that on 2.0 that same interactive interpreter would stop leaking making it viable in a greater range of circumstances.

Jim's got an open position for an SDE if you're interested in working on the future of dynamic typing at Microsoft.

Hard links and the GAC

While I was at Microsoft this week, Andrew Grealy put an interesting idea in my head: using NTFS hard links to emulate some of the functionality of the GAC. The scenario that I had in mind was to create NTFS hard links to duplicate private assemblies so that I could update the "shared" private assembly simply by replacing a single file on the user's hard drive.

On the plane ride home, I wrote a simple Python script to do just that. It looks for duplicate files on your hard disk, and moves a single copy of that file to a special place that we call the YAC (Yet Another Cache). It then replaces all copies of that file with a hard link that points to the YAC entry for that file.

It calculates MD5 hashes to ensure that duplicate files are in fact duplicates, and uses the Python Win32 extensions to call the Windows CreateHardLink API.

I absolutely love the use of Python Generators in the GetFiles method. This is such a low-friction language!


import os
import shutil
import md5
import win32file

class HardLink:
    BLOCKSIZE   = 1024
    YAC         = r'c:\dev\python\yac'
    
    def __init__(self):
        self.files = dict()
        
    def GetFiles(self, path):
        for root, dirs, files in os.walk(path):
            for file in files:
                yield root + '\\' + file

    def CalculateHash(self, path):
        f = file(path,'rb')
        m = md5.new()
        readBytes = HardLink.BLOCKSIZE
        while (readBytes):
            readString = f.read(readBytes)
            m.update(readString)
            readBytes = len(readString)
        f.close();
        return m.hexdigest()

    def GetYacPath(self, path):
        (dir, filename) = os.path.split(path)
        return HardLink.YAC + '\\' + filename

    def CreateFirstFileHardLink(self, path):
        yacPath = self.GetYacPath(path)
        shutil.move(path, yacPath)
        win32file.CreateHardLink(path, yacPath)

    def CreateNthFileHardLink(self, path):
        yacPath = self.GetYacPath(path)
        os.remove(path)
        win32file.CreateHardLink(path, yacPath)
        
    def Run(self, path):
        for path in self.GetFiles(path):
            hash = self.CalculateHash(path)
            if self.files.has_key(hash):
                (firstPath, flag) = self.files[hash]
                if not flag:
                    self.files[hash] = (None, True)
                    self.CreateFirstFileHardLink(firstPath)
                self.CreateNthFileHardLink(path)
            else:
                self.files[hash] = (path, False)

h = HardLink()
h.Run(r'c:\test')
