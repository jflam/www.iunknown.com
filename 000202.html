Consistency == usability

One of the hardest things to deal with when learning something new is inconsistency. I rely on finding patterns in systems and reducing those patterns to idioms that I can easily remember. There have been two different things that I have been working on recently that have driven me crazy and in both cases the root of the problem is inconsistency.


OS X applications lack consistency in the way they handle the keyboard. It seems like every application has a different set of keystrokes for extremely common operations. By common operations, I mean things that devs do every day: next word, previous word, beginning of line, end of line, top of document, bottom of document. Among the three editors that I use on a daily basis, email and two different programming editors, they all have different keystrokes for these simple operations. In one editor command left takes me to the beginning of a line, in another editor it takes me to the previous word. Because of this, my TiBook just feels ... awkward.


In the other case, I've been learning a new API. This API is incredibly inconsistent. The abstractions break down in so many different places. Just when you think you've got the API figured out, it throws yet another curveball at you. It makes me appreciate just how difficult it is to create an API that is actually usable, especially when it has to evolve over several product releases, and you have to maintain backward compatibility.
