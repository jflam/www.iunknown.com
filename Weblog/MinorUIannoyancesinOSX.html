Minor UI annoyances in OS X
2003-01-03

Here's a few things that I have found annoying about the OS X UI in the past few days. In no particular order:

  • Inconsistent usage of keyboard shortcuts. This one is driving me batty. For example, there are at least two different HTML browser-like things that I use. The first one is, of course, IE and I go backwards to the previous page using Command [. For the project builder documentation browser, I have no idea what the keyboard shortcut is. But it certainly isn't Command [. There are lots of places in the OS X UI where keyboard shortcuts are inconsistently used, which makes them so much harder to learn (another good example is Command M used for minimizing windows in every other application EXCEPT Acrobat Reader).
  • One thing that I really like about Acrobat Reader is its support for CoolType. This is about the closest thing to ClearType that I have found on the Mac. I have replaced the default PDF reader on OS X with Acrobat Reader for this reason alone. However, it's non-Mac-like interface really sticks out like a sore thumb.
  • Inconsistent mouse cursor changes to indicate what you can do. Again, using Project Builder as an example, if I'm over a documentation pane, the mouse cursor is an I-beam. When I move it over the main menu for the application, it remains an I-beam. If I click on a menu item, it becomes an arrow cursor (which is what it should have changed into when I moved my mouse over the menu). When I move my arrow cursor back over the documentation pane, it becomes an I-beam again. Why does it change going in one direction and not the other?
  • The fonts for the Cocoa documentation files are really small. Is there any way of globally changing the font sizes / font faces used to render the docs?
