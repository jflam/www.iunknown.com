VS.NET Schema Editor

I've been spending some time recently with the VS.NET XSD editor. Previously, I've generated data access layers from ad-hoc schema definitions in XML. I did this primarily to bootstrap the process and to partially to avoid the namespace issues in GSLgen. After my experience this afternoon, however, I'm on the verge of caving in and rewriting my code generation scripts to use the information in XSD schemas to [partially] generate the data access layers.


To be able to leverage XSD as-is would be awesome. Unfortunately, the current version (3.1) of GSLgen does not support namespaces. To further complicate matters, not only does it GSLgen not support namespaces, colons are reserved characters (identifiers in GSLgen cannot contain colons). This means that I cannot use xs:element as a valid element name in GSLgen. Grrrrrrr.


I might just run a sed/perl script over the XSD and convert all colons to underscores so that I can continue my work. But clearly, I would much prefer a world where GSLgen understood namespaces (and XPath too!).


While I'm busy complaining about my tools this afternoon (it really has been productive otherwise), I really wish the VS.NET folks would add an undo facility in the XSD editor. That and an auto-arrange that actually works would rock :)


Update: After spending a bit of time thinking about how easy / hard it would be to leverage XSD, I've decided to back off on the idea. Instead, I'll just port the XSD into my own ad-hoc schema language. I really need to generate Pet Shop first and worry about tool issues later.
