Feeling some VMWare 4.0 pain

Some time ago, I prepared a Win XP "baseline image" for VMWare 4.0 Workstation. I was under the impression that VMWare 4.0 disks started small and grew as you used them. This is true. But an ugly problem rears its head if you accepted the defaults when you created a new VM - the disk is set to a 4GB size limit AND YOU CANNOT CHANGE THIS AFTER THE FACT! Â 

So the moral of the story is to set a ridiculously large disk size when creating a new VM (128GB for IDE drives) so you can never run out of disk space (although you are obviously limited by the size of your physical HDD).

I am so not looking forward to creating yet another Win XP baseline image.

One way around it is to create a second virtual disk in your virtual machine of much larger size, use a tool like Ghost to copy the WinXP partition from the existing vdisk to the new vdisk, and then remove the first vdisk and let the virtual machine boot off the second vdisk.
The result should be a "new" baseline with a much greater capacity.
