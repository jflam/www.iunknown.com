Life with Win2K3

Life is good now that I have Win2K3 running on my laptop. Best feature? The ability to terminal server back to myself using different credentials. This makes it so much easier to run as a regular user without having to log out and log back in to do stuff like twiddle ACL's on files.

Some other cool things that I did:

  1. Create a custom MMC console that contains only the admin tools that I actually use. This is so much easier than searching through the Administrative Tools folder for a predefined MMC console.
  2. Configure Outlook 2003 to run in Cached Exchange mode.

Well, I had to repave a week ago, and decided to give Win 2003 a chance and finally make the switch to running as a non-admin. Things have mostly been going smoothly, although I did have to hack the registry to make Terminal Services find my local Licensing Server and not the domain-wide 2000 Server. I also had to *telephone* the Microsoft Clearinghouse in order to get an activation code that would work with our Certified Partner volume license.

Anyway, I am now temporarily back to running as an admin because I haven't found a reasonable solution to debugging ASP.NET web applications from Visual Studio.

I could let ASP.NET run under my user account, but I don't want to do that as that would reintroduce a mismatch between the development environment and the testing and production environments. That's a big deal, as the whole point of running as a non-admin (for me, anyway) is to eliminate such mismatches.

The other thing I am supposed to be able to do is to grant myself the SeDebugPrivilege, which supposedly is the same as turning on the "Debug programs" user right. This isn't ideal either, as I am slowly morphing into an administrator again, but it's not such a big deal, as the debugging privilege is not one that is likely to be a point of conflict when non-privileged users are running my applications. However, it doesn't work. I am logging out and back in, but I still get the "Access denied" message when trying to debug a web app.

The third way of solving this problem is to simply fire up Visual Studio from my admin Terminal Services client whenever I need to debug, but come on. That is far more of a productivity restriction than I am willing to accept.

So at the moment, I am back to running as an admin. I'd be very interested in hearing how others have tackled this problem. Are you all happy running ASP.NET under your personal accounts?
