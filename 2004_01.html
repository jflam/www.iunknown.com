Spit n' polish in ASP.NET 2.0

I've spent the better part of the day looking closer at ASP.NET 2.0. I'm really impressed by the amount of polish and attention to detail in the product. First Look at Venus

I spent some time this morning taking my first look at Venus (ASP.NET 2.0). The first thing that I wanted to pull apart was the new "Code Separation" compilation model that replaces "Code Behind" in ASP.NET 1.0/1.1.

Observation 1: Development uses the "Cassini" managed web server to host ASP.NET, not IIS. This is great for a number of reasons. Perhaps most important, I can now do ASP.NET development under a restricted account without having to jump through various hoops.

Observation 2: The code that ASP.NET generates is no longer in the usual place. Instead, you can find it under Local Settings\Temp\Temporary ASP.NET Files\... A big thanks goes out to Mark Russinovich's most excellent Filemon tool.

Observation 3: ASP.NET now uses the partial classes feature of C# for all user-defined code. When you write event handlers, the only code that you see is the code that handles the event. None of that nasty region-hidden code ever appears.

Observation 4: Intellisense works right away in the partial class file. This is really cool. Consider the following code frag:


using System;

namespace ASP
{
  public partial class Default_aspx
  {
    void Button1_Click(object sender, System.EventArgs e)
    {
      Label1.Text = "Clicked, me!";
    }
  }
}

When I type Label1 I see the Intellisense right away for the Label1 control. This requires a lot of background compilation goodness. Kudos to the IDE team for making this such a painless process.

Observation 5: Intellisense works in the .aspx file. Now, most of the time I don't see a good reason to put code in the .aspx file, but in those cases where you need to do this (customization at a customer site comes to mind) it sure helps having Intellisense in your <script runat="server"> blocks.

More observations later, but so far I'm really liking what I see.

January 04, 2004

Ant pushback

I recently came across two rants about Ant. These rants were written by folks that I respect, and I'm quite disappointed that they didn't post more reasoned/detailed arguments.

The first, written by Bruce Eckel a confusing rant at best. I almost couldn't find his references to Ant amidst some reminiscing about C++ and the origins of Java. Bruce doesn't like Ant because: the file format is XML, and developers shouldn't be forced to have to write verbose build.xml files by hand. To me, this is largely an IDE problem as opposed to a problem with Ant as a build tool in and of itself. It isn't clear from looking at his rant what scenarios he had in mind when he wrote that "Ant sucks on a number of levels". It would be a much more informed discussion if he could point out specific deficiencies in Ant that caused him problems. I'm sure that the community would be more than happy to help him succeed with Ant.

The second, written by Martin Fowler is not particularly dismissive of Ant. Instead, he focuses on how conditionals and declarative languages don't mix. He proposes using an extension to a full-blown programming language, Ruby, as a replacement for Ant. I'd be interested in seeing some specific scenarios where he requires the power of a full-blown programming language to implement a build script.

Random Walk Guide to Investing

Earlier I recommended a book called The Intelligent Asset Allocator. Today, I just finished a reading a far more approachable book that covers the same ground: The Random Walk Guide to Investing by Burton Malkiel:

I highly recommend reading this book before tackling Bernstein's book. It's a good mix of personal finance guidance with practical portfolio balancing advice. He also presents a much more simplified view of an investment portfolio than Bernstein's book. For example, in the equity portion of your portfolio, Malkiel advocates using extremely broad market index tracking funds like the Vanguard Total Stock Market Index fund that tracks the Wilshire 5,000 index. Contrast that with Bernstein's recommendation for a mix of large-cap, small-cap and foreign stocks (and that's his Level 1 (a.k.a. simplest) Asset Palette)!

January 03, 2004

Is the browser finally being relegated to its rightful place?

Yahoo News reports that Internet Applications are the primary way that people connect to the Internet. I think it's a good thing that the web is being relegated to being mostly an information browsing resource. Even that is being threatened since most of my information comes to me in the form of RSS feeds these days.

Applications like Quicken and Money, which I'm using a lot these days, sport a web interface mostly because it allows the authors of those programs to provide a rich UI. However, I've run into a number of usability issues that could easily go away if financial institutions lessen their dependency on providing information using a web browser. For example, downloading statements to Money is complicated by the fact that I have to first log into the bank's web site before I can do anything else. In the day and age where that information is only a Web Service call away, the need to interact with my bank through a browser will finally go away.

It will be very interesting to see what a personal finance application circa 2006 will look like. On Windows, it will most likely sport a XAML front end that calls via Indigo against Web Services provided by the various financial institutions. I suspect that news feeds will be provided by some sort of RSS-like mechanism so that the application can render the news in such a way that it looks consistent with the rest of the application. It will be an "Internet Application", but one that will likely not use a browser at all. I can't wait.

New Years Reflections

Happy New Year! What a change this year has been for us! Here's a brief recap of what's happened in our family this year:

Kobe joined our family. In a way, Kobe was our "training baby" and he really helped us prepare for the larger changes that awaited us.

Matthew joined our family. It really is amazing to see the changes in him as the weeks go by. This week, we noticed that he's getting better at sitting up and looking around at the world. He's beginning to grasp and reach for objects that are close to him, and he's got a new favorite toy: froggy (or as Carolyn likes to say: Fwoggy).

We bought a new house to accommodate our growing family, and are counting down the days that we remain in our apartment. I've lived in the same apartment now for nearly 6 years. We've been very happy here - the views are fantastic, and the staff in the building is awesome. We're going to miss life here, but at the same time, we're really looking forward to having some more space.

All of these things forced me to grow up in ways that I hadn't really imagined. The first thing that I had to fix was my health: I decided that I was through with being a fat guy. So on June 6, I decided to change the way that I ate. Most of my inspiration was drawn from the book "Enter the Zone":

The key thing about this book was that it got you to count something other than calories. It also appealed to me intellectually since the book spends a lot of time explaining the biochemical responses that we have towards different food types. While his discussion on eicosanoids is somewhat suspect, the core message in the book: carbohydrates should be treated as a condiment, really resonated with me.

I'm happy to report that I'm back to within 2 pounds of my college weight - a total loss of 42 pounds since June. Most of that weight came off within the first 3 months or so, and I've been on maintenance for the past two months or so. The disappearance of my tummy has also helped during those times when you're trying to do work in a coach airplane seat and the guy in front decides to put his seat back :)

The second thing that starting a new family has made me think about is the state of our personal finances. We truly are very fortunate to have the income that we have, especially considering some of the dark clouds that hang over the software / IT industry at large. That said, however, I've really been ignorant of the state of our personal finances.

So, this holiday I decided to educate myself. My first steps were to read some books on personal financial management. The first book, The Millionaire Next Door, was a real eye-opener:



p. The take-home message in this book is to pay yourself first. Watch your expenses. And spend some time thinking about how you can / should invest the money that you paid yourself. Now, this formula is tried-and-true. Virtually every self-help personal finance book will tell you some variation on this theme. However, I really connected with the way that this book tells the story, particularly the way he groups individuals into the UAW (under accumulator of wealth) and the PAW (prodigious accumulator of wealth) categories. Hopefully, this year marks the beginning of my transition from the UAW to the PAW category. Note that this doesn't necessarily have anything to do with increasing income - just doing different things with the income that we already have.

Along the way, I tried out a few pieces of personal financial software. First, I went out and purchased a copy of Quicken XG since it's the market leader. I quickly became annoyed by the fact that I had to grant write privileges to the program's directory under \Program Files. Next, product activation got in the way when I wanted to run Quicken on my laptop instead of on my wife's desktop.

In frustration, I went out and bought a copy of Money 2004 Canadian Edition. So far I've been very happy. It does everything that I want, it runs fine under a non-privileged user account, it costs 1/3 of what Quicken XG does, and I can run it on whatever computer that I want. The only annoying thing is the fact that I have to save the .ofx file that my bank transmits to me to disk before importing it into Money. Under Quicken, this is a seamless operation (e.g. click on the Download button and the information shows up in Quicken - no manual import step). I look forward to the day when we can dispense with the download option entirely and simply use Web Services to retrieve the account information from the bank.

With the record keeping stuff out of the way, I'm now turning my attention towards taking an active role in managing our investment portfolio. One of the most interesting books that I've come across is the Intelligent Asset Allocator by William Bernstein:

The premise of his book is that the returns on an investment portfolio over the long term have more to do with the composition of the portfolio instead of the selection of the individual investments. This argues strongly for using index tracking funds to hold the investment categories, since they have much lower management fees than "managed" funds - particularly if you believe his assertion that stock picking is a random walk with no potential for long-term success. It's an interesting thesis, he makes a very good argument and backs it up with a large amount of data. Highly recommended if you want to take a hard look at the performance of your own investment portfolio.
