Sheesh - it's hard to give away code these days!

I just spent some time uploading an early Alpha build of CodeRef into a GotDotNet Workspace. I was looking forward to a better user experience than SourceForge. However, I was disappointed. Here, in no particular order are some of the issues:

  • Horrible performance using the Source Control (WinForms control embedded into IE) option. When the control initializes, it ties up the UI thread of IE to communicate with the mothership. This pretty well locks up IE entirely while it does whatever it needs to do to initialize itself.
  • I can only successfully connect maybe 25% of the time today to my workspace using the Source Control. I tried it on two different computers here, and I have roughly the same success rate on each one.
  • No command line interface. This makes it much harder for me to integrate GDN source control into my existing NAnt-driven build system for CodeRef.

I really don't want to host my open source projects using SourceForge either (and to make things worse, I can't connect right now to SourceForge either). I might spend a bit of time looking at setting up a public Perforce server to host CodeRef instead, but I really want all of the other features that GDN and SF offers - bug tracking, mailing lists etc.

Why do things have to be so difficult?!!
